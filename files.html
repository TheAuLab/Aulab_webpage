    Quantifying the expression of transcripts by salmon version 1.0.0 and output count matrix by R



    Input files:


    • Annotation (Annotation is only for version higher than 1.0 plus versions. OSC installed salmon v.1.0.0. The latest version of salmon is 1.2.0. In this case, we only use version 1.0.0):

      •             
            wget ftp://ftp.ensembl.org/pub/release-99/gtf/homo_sapiens//Homo_sapiens.GRCh38.99.gtf.gz

    • Transcriptomes (Ensembl):

      •             
            wget ftp://ftp.ensembl.org/pub/release-99/fasta/homo_sapiens//cdna/Homo_sapiens.GRCh38.cdna.all.fa.gz

    • Reads (you can quantify any set of reads, say paired-end reads): 

      •             
            X5608_GuoD_GBM30-con1_V1N_1_S1_L001_R1_001.fastq.gz
      
            X5608_GuoD_GBM30-con1_V1N_1_S1_L001_R2_001.fastq.gz


    Running Salmon 1.0.0:


                
            module load salmon/1.0.0


    Prepare index for Salmon:


                
            salmon index -t Homo_sapiens.GRCh38.cdna.all.fa -i transcripts_index


    Quantifying:


                
            salmon quant -i transcripts_index -l A -1 X5608_GuoD_GBM30-con1_V1N_1_S1_L001_R1_001.fastq -2 X5608_GuoD_GBM30-con1_V1N_1_S1_L001_R2_001.fastq  --validateMappings -o /fs/project/PCON0005/data/DLG_RNA-seq/191018-transfer/X5608_transcripts_quant
    Notice: " -l/libType " means library type; "A" allows salmon automatically infer library type for you; see more detail on salmon official website.


    Output files:


    For each read, salmon will output a folder. The files in that folder are showed below:
                
            aux_info  cmd_info.json  lib_format_counts.json  libParams  logs  quant.sf

    We need to collect quant.sf file for read counts. This is what quant.sf file looks like. The last column is read counts. 
                
            Name    Length  EffectiveLength TPM     NumReads
            ENST00000434970.2       9       10.000  0.000000        0.000
            ENST00000415118.1       8       9.000   0.000000        0.000
            ENST00000448914.1       13      14.000  0.000000        0.000

    Download all the quant.sf files for your reads, put all sf files into a new folder, in this case “quant”. Open RStudio to combine all read counts, R code is below: 
                
            dir <- "/Users/yuruwang/Desktop/quant" # path of files 
    
            filename <- list.files(dir) # list files in this path 
    
            files <- file.path(dir, filename) # store path for each file 
    
            all(file.exists(files)) # check files exists or not 
    
            lst <- lapply(files, read.table, header=TRUE, stringsAsFactors=FALSE) 
    
            # store files into a list 
    
            
            count <- lapply(lst, "[", c("Name", “NumReads”)) # keep only ID and read counts 
    
            res <- do.call(cbind, count) # because the order of ID are same for each sf file, combine the columns of each file. 
    
            colnames(res)[1] <- “Isoform" 
    
            r <- res[,-grep("\\Name$", colnames(res))] # remove all ID column except the first column(Isoform ID) 
    
            colnames(r)[-1] <- filename 
    
            write.csv(r,file = "all_counts.csv",row.names = FALSE)

    You can then use DESeq2 or Limma-Voom do differential expression analysis. They are R packages. Please take a look of DESeq2 and Limma-Voom examples.

    Manual for construction of co-expression networks using Iso-Net


    This tutorial will help you infer co-expression networks using Iso-Net based on exon-level expression data. If you experience any problems following these steps, please don't hesitate to contact Dingjie Wang.


    Step 1: Requirements


    (1) System Requirements: Windows or Linux operating system, and R 3.6.1 or later

      Update R on Windows 10:
        1. Open your R, type these code in console:
        install.packages("installr"); library(installr)
        updateR()
        2. Or you can type only install.packages("installr"); library(installr) and then click the installr menu. Choose "Update R".

      Update R on Mac:
        On Mac, you can simply download and install the newest version of R. When you restart RStudio, it will use the updated version of R.

    (2) R package required: CompQuadForm and survey



    Step 2: Download lastest version of Iso-Net


    In order to run Iso-Net, you should first fetch the source code from Iso-Net git repository.

    $ git clone --recursive https://github.com/Dingjie-Wang/Iso-Net.git
    $ cd Iso-Net



    Step 3: Run Iso-Net


    Open R environment in Windows or Linux operating system, type the following commands:

    $ source("IsoNet.R");
    $ PerformIsoNet("Input_file.txt","output_file.txt",sample_size);
    Here, "Input_file.txt" is the name of input file, "output_file.txt" is the name of result file and sample_size if the sample size of data.



    Step 4: The format of input file and the output file


    You can try the example files. These files are provided by Dingjie Wang.

    Input file: In this file, the first column is the gene ID, the second column is the exon ID, and the expression of samples are from the third column to the last column.
    Output file: In this file, each row represents an possible edge. The first and second column are the two genes' IDs. For RVNet method, it includes three columns and the third column is the RV cofficient. For MINet method, it includes four columns, the third column is the test statistics of the possible edge and the last column is the p-value.
